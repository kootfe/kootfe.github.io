Dumb Linux Commands

Jan 03, 2026 | Koofte

  • tac:

    Unlike cat, which reads a file top to bottom, tac reads it from bottom to top.

  • watch:

    Reruns a command every X seconds, useful for monitoring changes in real time.

  • tee:

    Like > or >> but more advanced: reads from stdin and writes to both a file and stdout simultaneously.

  • wc:

    Counts characters, words, and lines in a file.

  • join:

    Merges two sorted files line by line.

  • realpath:

    Prints the absolute path of a file. (~/file → /home/user/file)

  • basename:

    Outputs the filename from a path. (/path/to/file.mp69 → file.mp69)

  • dirname:

    Opposite of basename: returns the directory path. (/path/to/file → /path/to)

  • more:

    Displays file contents one screen at a time. Less is generally better.

  • cal:

    Displays a calendar.

  • fmt:

    Wraps text to a specified width, preserving words. Maximum X characters per line.

  • factor:

    Breaks numbers into their prime factors.

  • nice:

    Sets the priority of a command. Usage: nice -n priority command; Lower the number → higher priority. Only root can use negative niceness. Example: nice -n 5 discord && nice -n 15 steam Discord gets higher CPU priority than Steam.

  • sort:

    Sorts lines in a file. Options: -h for human-readable (sizes), -u for unique lines.

  • comm:

    Compares two sorted files line by line.

  • uniq:

    Removes duplicate lines and optionally counts occurrences.

  • mktemp:

    Creates temporary files in /tmp with unique names. Does not auto-delete.

  • xargs:

    Takes output from one command and passes it as arguments to another. Example: echo "file1 file2" | xargs rm Unlike a simple pipe, some commands (like rm) need arguments, not stdin input.

  • yes:

    Prints "yes" (or a custom string) repeatedly. Useful for automating inputs or stress-testing CPU.

  • pv:

    Pipe viewer: shows progress of data through a pipeline.
